# PMS 小程序项目说明

## 项目结构

- **pms-tenant**：租客端小程序（uni-app Vue3 + TypeScript）
- **pms-staff**：物业员工端小程序（uni-app Vue3 + TypeScript）
- **pms-web**：后端 API（已新增 `/api/mp/*` 接口）

## 启动方式

### 1. 启动后端 pms-web

```bash
cd pms-web
npm run dev
```

后端默认运行在 `http://localhost:5000`。（若 5000 端口已被占用，说明后端可能已在运行）

### 2. 启动租客端 pms-tenant

```bash
cd pms-tenant
npm install --legacy-peer-deps
npm run dev:mp-weixin   # 微信小程序
```

编译成功后，使用**微信开发者工具**打开 `pms-tenant\dist\dev\mp-weixin` 目录。

### 3. 启动员工端 pms-staff

```bash
cd pms-staff
npm install --legacy-peer-deps
npm run dev:mp-weixin
```

编译成功后，使用**微信开发者工具**打开 `pms-staff\dist\dev\mp-weixin` 目录。

## 后端新增 API

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/mp/login` | POST | 小程序登录，body: `{ phone, password, type: 'tenant' \| 'employee', companyId?, tenantUserId? }`。租客端同一手机号可在多家物业公司、多个租客下有多条**独立**账号；与后台「员工」表无关。跨公司时需 `companyId` 或 `needCompany`；同一公司下多条租客账号时需 `tenantUserId` 或 `needTenantUser`。 |
| `/api/mp/me` | GET | 获取当前用户（需 `Authorization: Bearer <token>`） |
| `/api/mp/bills` | GET | 租客账单列表 |
| `/api/mp/work-orders` | GET/POST | 租客/员工工单（租客可创建） |
| `/api/mp/announcements` | GET | 公告列表 |
| `/api/mp/complaints` | GET/POST | 租客卫生吐槽 |
| `/api/mp/my-todos` | GET | 员工待办（工单+巡检） |
| `/api/mp/inspection-tasks` | GET | 员工巡检任务 |
| `/api/mp/my-messages` | GET | 员工消息（物业公告 + 报事报修/巡检/卫生吐槽等业务通知，按时间合并） |
| `/api/mp/inspection-tasks/[id]` | GET | 员工巡检任务详情 |
| `/api/mp/complaints/[id]` | GET | 员工卫生吐槽详情 |

## 员工业务通知（楼宇 + 管理业务类型）

- 租客提交**报事/报修**、**卫生吐槽**，或系统生成**巡检任务**时，会向「该楼宇所属项目」下、且后台员工档案中勾选了对应**管理业务类型**的员工写入站内通知（如报修→`报修`，吐槽→`保洁`，巡检类型与 `工程/安保/设备/绿化` 等对齐）。
- 员工需在 Web **员工管理**中绑定**所属项目**（项目的楼宇范围需包含目标楼宇），并勾选相应业务类型，否则收不到通知。
- 数据库表：`StaffNotification`（执行 `npx prisma migrate deploy` 或 `db push` 同步）。

## 租客账号规则（与 Web「员工管理」无关）

- **后台员工（Employee）**：全系统手机号唯一，仍按原逻辑。
- **租客端员工（TenantUser）**：不按物业公司做手机号唯一；同一手机号可在**同一物业公司下的多个租客**下各添加一次（每次一条独立账号）；同一租客下同一手机号仍不可重复。
- 从旧库升级：请在 `pms-web` 执行 `npx prisma db push`（已去掉租客表上「手机号+公司」唯一索引的需同步数据库）。

## 配置说明

- 小程序请求的 `baseURL` 在 `src/api/request.ts` 中配置，默认 `http://localhost:5000`
- 微信小程序需在「详情 - 本地设置」中勾选「不校验合法域名」以便本地调试
- 发布前需将 `baseURL` 改为正式服务器地址
